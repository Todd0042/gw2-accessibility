┌─ OVERVIEW ───────────────────────────────────────────────────────────────┐

A Guild Wars 2 accessibility addon (DLL) loaded by the Nexus addon loader.
Provides auditory (TTS), visual (boon/condition overlays via Reffect), and
cognitive (keybind display, mouse position memory) features.

   File:      build/GW2Accessibility.dll
   Author:    Todd0042
   Version:   1.1.0 (CMakeLists.txt) / 0.2.3.0 (Nexus API, addon.cpp:4352)
   License:   Not specified

   AddonLoad flow:
     1. DetectWine() — checks for wine_get_version in ntdll.dll

     3. EnumerateTtsVoices() — scans HKLM registry for TTS voice tokens
     4. EnumerateTtsOutputDevices() — queries waveOutGetDevCaps for audio devices

┌─ TTS SYSTEM (Windows) ───────────────────────────────────────────────────┐

   Architecture: Windows SAPI5 via COM

   InitSAPI() (line 2259):
     1. CoInitializeEx (tries MT first, falls back to STA)
     2. CoCreateInstance(CLSID_SpVoice) → g_pVoice (ISpVoice*)
     3. Voice: creates ISpObjectToken, calls SetId() with full HKLM path
     4. Output: CoCreateInstance(CLSID_SpMMAudioOut, IID_ISpMMSysAudio)
        then ISpMMSysAudio::SetDeviceId(waveIndex), then SetOutput()
     5. SetVolume(0-100), SetRate(-10 to 10)
     6. Logs active voice description

   Voice enumeration — EnumerateTtsVoices() (line 2192):
     Scans two HKLM registry paths for voice tokens:
       HKLM\SOFTWARE\Microsoft\Speech_OneCore\Voices\Tokens
       HKLM\SOFTWARE\Microsoft\Speech\Voices\Tokens
     Each token's (default) value = display name.
     Token ID = full registry path string.
     First entry is always "System Default" (empty tokenId).

   Output device enumeration — EnumerateTtsOutputDevices() (line 2235):
     Uses waveOutGetNumDevs() + waveOutGetDevCapsW() to list audio outputs.
     Token ID = stringified wave device index ("0", "1", etc.) or "" for default.
     First entry is always "Default (System)" (empty tokenId).

   Speaking — SpeakText() (line 334):
     Calls g_pVoice->Speak(text, SPF_ASYNC | SPF_IS_NOT_XML).
     If SAPI fails and Wine is detected, falls back to SpeakViaWinePipe().

   TTS settings persisted:
     g_Tts.enabled        — master toggle
     g_Tts.party/squad/whisper/etc.  — per-channel toggles (13 channels)
     g_TtsVoiceTokenId    — full HKLM path of selected voice
     g_TtsOutputTokenId   — wave device index string
     g_TtsVolume          — 0-100
     g_TtsRate            — -10 to 10

┌─ TTS SYSTEM (Linux/Wine) ───────────────────────────────────────────────┐

   On Wine/Proton, SAPI is a stub. The addon detects this via DetectWine()
   and falls back to a named pipe → native Linux TTS engine.

   Flow:
     1. InitSAPI() — CoCreateInstance(CLSID_SpVoice) FAILS
     2. g_WineTtsFallback = true (addon.cpp:2368)
     3. SpeakText() fails → SpeakViaWinePipe() (addon.cpp:155)
     4. Writes UTF-8 text to ~/.gw2-tts-pipe (via Wine Z:\ drive)
     5. tts_helper.sh reads pipe and calls espeak-ng or Piper

   Pipe path: Z:\<HOME>\.gw2-tts-pipe  (addon.cpp:140)
   Config:    ~/.gw2-tts-config (read by tts_helper.sh)

   tts_helper.sh features (tts_helper.sh):
     ENGINE    — "espeak-ng" (default) or "piper"
     VOICE     — espeak-ng voice name or Piper model tag
     SPEED     — espeak-ng words per minute (default 150)
     PITCH     — espeak-ng pitch (default 40)
     AMPLITUDE — espeak-ng volume (default 175)
     Auto-detects Piper binary in several common locations.
     Searches ~/.local/share/piper-voices for Piper model files.
     Creates a named pipe at ~/.gw2-tts-pipe.
     Safe to run multiple times (cleans up stale pipe on start).

   Settings that do NOT work on Linux (no SAPI):
     Voice dropdown         — HKLM registry empty on Wine, only "System Default"
     Output device dropdown — waveOutGetDevCaps returns 0, only "Default (System)"
     Volume slider          — g_TtsVolume stored but pipe ignores it
     Speed slider           — g_TtsRate stored but pipe ignores it
     These are controlled in ~/.gw2-tts-config instead.

   The "Test TTS" button in settings WILL work on Linux if the pipe daemon
   is running — it triggers InitSAPI() which fails, then SpeakViaWinePipe().

┌─ REFFECT PACK INTEGRATION ──────────────────────────────────────────────┐

   The addon contains an embedded Reffect UI pack definition that it deploys
   to the Reffect addon's packs directory on startup.

   File:         src/reffect_pack.h (auto-generated from GW2Accessibility.json)
   Source:       GW2Accessibility.json (project root, 2510 lines)
   Target:       <reffect addon dir>\packs\GW2Accessibility.json

   Versioning:   "packVersion" field inside the JSON
                 On deploy, if on-disk version >= embedded version, skip.
                 If no version found (old file), always overwrite.
                 Manual deploy button in Settings → Visual section.

   DeployReffectPack() (addon.cpp:678):
     1. GetReffectPacksDir() — resolves path via Nexus API or fallback
     2. If file exists and on-disk version >= embedded, skip
     3. Otherwise, overwrite with embedded g_ReffectPackJson

   Update procedure when GW2Accessibility.json changes:
     1. Bump "packVersion" by 1 in GW2Accessibility.json
     2. Re-embed: run the PowerShell command from AGENTS.md
     3. Rebuild

┌─ COMMON TASKS ───────────────────────────────────────────────────────────┐

   Add a new setting:
     1. Add variable at top of addon.cpp in the relevant section
     2. Save it in SaveSettings(), load it in LoadSettings()
     3. Add UI control in OnOptionsRender()

   Update the Reffect pack:
     1. Edit GW2Accessibility.json
     2. Bump "packVersion" by 1
     3. Re-embed: run the one-liner in AGENTS.md
     4. Rebuild

   Build & deploy:
     cmake -B build -DCMAKE_BUILD_TYPE=Release
         "-DCMAKE_TOOLCHAIN_FILE=toolchain-mingw.cmake"
     cmake --build build --config Release
     Copy build/GW2Accessibility.dll to <Nexus>/addons/GW2Accessibility/

   Fix Linux TTS:
     - Ensure tts_helper.sh is running (ps aux | grep tts_helper)
     - Check pipe exists (ls -la ~/.gw2-tts-pipe)
     - Test manually: echo "hello" > ~/.gw2-tts-pipe
     - Configure voice in ~/.gw2-tts-config
     - See TTS_SETUP.md for detailed setup

   Debug logging:
     All TTS operations log via g_API->Log(LOGL_INFO, "GW2Accessibility", ...)
     Look for [TTS] prefix in Nexus log output.
